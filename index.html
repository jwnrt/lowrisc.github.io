Open to the core
Enabling open source silicon
through collaborative engineering

  • Open

    Our designs are permissively licensed and developed with collaborators from around the world. We implement the free and open RISC-V ISA.

  • Secure

    We are pursuing novel security features such as tagged memory. Unlike proprietary offerings, all aspects of the design can be fully audited.

  • Flexible

    Hardware needs softening up - through minion cores we enable I/O interfaces and real-time control to be defined in software.

lowRISC: a not-for-profit community interest company with an engineering team based in Cambridge, UK

developing and maintaining high quality, verified open source silicon IP

creating open source software tools, including compiler infrastructure and test and verification systems

an independent non-profit home for projects with multiple partners

research collaborations with universities to explore novel features in security and more

bringing the benefits of open source processes, community, and licensing to the hardware world

enabling open hardware to be a reality for diverse applications at scale

    The RISC-V LLVM backend in Clang/LLVM 9.0

    On Monday I proposed promoting the upstream RISC-V LLVM backend from “experimental” to “official” for the LLVM 9.0 release. Responses so far are extremely positive, and we’re working to ensure this is a smooth process. This means that from 9.0, the RISC-V backend will be built by default for LLVM, making it usable out of the box for standard LLVM/Clang builds.
